Chapter 5 — Sets

5.1 Membership and Extension

I. Concept

  • A set is a well-defined, unordered collection of distinct objects.
  • Membership: x ∈ S; non-membership x ∉ S ⇔ ¬(x ∈ S).
  • Extension: list elements in braces, e.g. s == {a, b, c}.
  • Syntactic abbreviation: n == e — n is a name for e, hence equal.
  • Axiom of extension: two sets equal iff same members; order and duplicates irrelevant.
  • Special sets: ℕ = {0, 1, 2, 3, …}; empty set ∅ has no members.
  • Subset: s ⊆ t ⇔ every element of s is in t; s ⊆ t ∧ t ⊆ s ⇔ s = t.
  • No size restriction: sets may be finite, infinite, or even undecidable.

5.2 Set Comprehension

I. Concept

  • Set comprehension: {x : s | p} = elements x from s satisfying predicate p.
  • Declaration (generator): x : s ranges x over set s.
  • Predicate (filter): the condition p that selects elements.
  • With term: {x : s | p • e} = set of expressions e for each x satisfying p.
  • Short forms: {x : s | p} = {x : s | p • x}; {x : s • e} = {x : s | true • e}.
  • Multi-variable: {x : a; y : b | p • e} ranges over two sets.
  • Bound variables may be renamed without changing meaning.
  • Supported by axiom of specification (predicate) and axiom of replacement (term).

5.3 Power Sets

I. Concept

  • The power set &Pset; a is the set of all subsets of a.
  • Size: if |a| = n, then |&Pset; a| = 2n.
  • Always includes: ∅ (empty set) and a itself.

  • Finite power set: &Fset; a = only the finite subsets of a.
  • Typing: &Pset; T used to declare “a set of T” in specifications.

5.4 Cartesian Products

I. Concept

  • The Cartesian product a × b = all pairs (x, y) with x ∈ a, y ∈ b.
  • An ordered pair (x, y) has position: (x, y) ≠ (y, x) in general.
  • An n-tuple (x₁, …, xₙ) generalizes pairs to n components.
  • [cart-mem]: (x₁, …, xₙ) ∈ a₁ × … × aₙ iff each xᵢ ∈ aᵢ.
  • [cart-eq]: tuples equal iff all components equal.
  • Projection: t.1, t.2, etc. extract components.
  • Size: |a × b| = |a| · |b|.
  • Order matters: a × b ≠ b × a unless a = b.

5.5 Union, Intersection, and Difference

I. Concept

  • Union: a ∪ b = elements in a OR b; [union]: x ∈ a ∪ b ⇔ x ∈ a ∨ x ∈ b.
  • Generalized union: ⋃ s for a set of sets; [Union]: x ∈ ⋃ s ⇔ ∃ a : s • x ∈ a.
  • Intersection: a ∩ b = elements in a AND b; [inter]: x ∈ a ∩ b ⇔ x ∈ a ∧ x ∈ b.
  • Generalized intersection: ⋂ s; [Inter]: x ∈ ⋂ s ⇔ ∀ a : s • x ∈ a.
  • Set difference: a \ b = elements of a NOT in b; [diff]: x ∈ a \ b ⇔ x ∈ a ∧ x ∉ b.
  • Union axiom: ZF guarantees ⋃ s exists.
  • Properties: ∪ and ∩ are commutative and associative; \ is not commutative.

5.6 Types

I. Concept

  • In Z, every value has exactly one type — the largest set s for which x ∈ s.
  • Built-in type: ℤ (all integers).
  • Basic type: declared with [T], e.g. [Person] — no visible internal structure.
  • Derived type: constructed via &Pset; and × from basic types.
  • Type checking: algorithms verify consistency of variable usage; x ∈ s valid only if type of s = &Pset;(type of x).
  • Types prevent Russell’s paradox: R == {s : T | ¬ s ∈ s} is type-blocked.
  • Choose basic types large enough: prefer all people over just UK nationals.

V.21 Practice: Basic Type Choice

When choosing a basic type for a system, you should:

Correct. e.g., use all Person, not just UK nationals.

  • Value: entities to model.
  • Type: pick a superset.
  • Check: avoid restricting too early.
  • Consequence: flexibility in specification.

V.22 Practice: Empty Set Type

What’s the type of ∅?

Correct. ∅ must be given a type by context.

  • Value: ∅.
  • Type: needs context (e.g. &Pset; ℕ, &Pset; Person).
  • Check: Z requires explicit typing.
  • Consequence: type determines what ∅ means.

V.23 Practice: Grade Function Type

Type “each student has one grade per course.”

Correct. Function from (student, course) pair to a number.

  • Value: grade assignment.
  • Type: Student × Course → ℕ.
  • Check: compound derived type.
  • Consequence: each pair maps to exactly one grade.

V.24 Practice: Pair Set Type

The type of {(1, a), (2, b)} is:

Correct. Set of pairs = power set of product.

  • Value: {(1,a), (2,b)}.
  • Type: &Pset;(ℕ × Letter).
  • Check: each element is a pair.
  • Consequence: relation type.

V.25 Practice: Multiset Type

Type “a multiset of tags” (tags with counts).

Correct. A bag/multiset maps each tag to its count.

  • Value: tags with multiplicities.
  • Type: Tag → ℕ.
  • Check: function type = derived.
  • Consequence: &Pset; Tag would lose counts.
