Chapter 22 — A Bounded Buffer Module

22.1 Specification

I. Concept

  • Bounded buffer: FIFO queue with fixed capacity — values leave in arrival order.
  • Abstract state: Buffer[X] has buffer : seq X, size : ℕ, max_size : ℕ with invariant size = #buffer ∧ size ≤ max_size.
  • Generic parameter: X is the type of stored values — the buffer is reusable for any type.
  • Three operations: BufferInit (initialise), BufferIn (insert), BufferOut (extract).
  • Partial ops: BufferIn₀ requires size < max_size; BufferOut₀ requires buffer ≠ ⟨⟩.
  • Report free type: Report ::= ok | full | empty; Success emits report! = ok.
  • Total ops: BufferIn ≅ (BufferIn₀ ∧ Success) ∨ BufferInError; precondition = true.
  • UpdateBuffer: includes Buffer and Buffer′ with max_size′ = max_size — capacity fixed.

22.2 Design

I. Concept

  • Circular array: design refines the abstract buffer into a fixed-length array with two index pointers.
  • Five components: array : seq X, max_size : ℕ, bot : ℕ (first live value), top : ℕ (last live value), size : ℕ (count of live values).
  • Invariant: bot, top ∈ 1..max_size; size ∈ 0..max_size; #array = max_size; size mod max_size = (top − bot + 1) mod max_size.
  • Why size needed: empty (size = 0) and full (size = max_size) both satisfy top + 1 mod max_size = bot — index collision is ambiguous.
  • Wrap-around: next index after max_size is 1; modular arithmetic governs pointer movement.
  • UpdateArray: includes Array and Array′ with max_size′ = max_size — capacity pinned at design level too.
  • O(1) operations: insert advances top; extract advances bot — no element shifting needed.

22.3 A Retrieve Relation

I. Concept

  • Retrieve relation: maps concrete Array to abstract Buffer — tells us which abstract state a concrete state represents.
  • Shift operator «: rotates a sequence left by n positions; axioms: n « ⟨⟩ = ⟨⟩, 0 « s = s, 1 « (⟨x⟩ ⌢ s) = s ⌢ ⟨x⟩.
  • Retrieve formula: buffer = (1..size) ◁ ((bot − 1) « array) — shift so bot lands at position 1, then keep the first size elements.
  • Functional refinement: the retrieve is a total, surjective function → we can calculate (not just verify) concrete operations.
  • Calculated concrete ops: ArrayIn₀: size′ = size + 1; bot′ = bot; top′ = (top mod max_size) + 1; array′ = array ⊕ {top′ ↦ x?}.
  • Calculated ArrayOut₀: size′ = size − 1; bot′ = (bot mod max_size) + 1; array′ = array; x! = array bot.
  • Error schemas: ArrayInError simplifies to size = max_size ∧ ΞArray ∧ report! = full; ArrayOutError to size = 0 ∧ ΞArray ∧ report! = empty.
  • Key identity: #((1..size) ◁ ((bot − 1) « array)) = size — proved by calculation using properties of restriction and shifting.
