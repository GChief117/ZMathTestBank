Chapter 8 — Functions

8.1 Partial Functions

I. Concept

  • Function: a single-valued relation — each input maps to at most one output.
  • Partial function: X ⇸ Y — some elements of X may have no mapping in Y.
  • Total function: X → Y — dom f = X; every source element is mapped.
  • Hierarchy: total function ⊆ partial function ⊆ relation.
  • Function application: f(a) = the unique y such that a ∈ dom f.
  • app-intro rule: if unique pair in f with first = a, and a↦b ∈ f, then b = f(a).
  • app-elim rule: if b = f(a) and unique pair with first = a, then a↦b ∈ f.
  • Active badge example: where_is ∈ Person ⇸ Location — partial because off-site employees have no location.

IV.5 Examples: Type Hierarchy, Classification Ladder & Subset Chain

SymbolPlain English

A software architect explains how types nest: every total function is automatically a partial function too.

  • Type: (X → Y) ⊆ (X ⇸ Y) — total is a subset of partial.
  • Domain: a total function has dom f = X; a partial function may not.
  • Uniqueness: both are single-valued — the only difference is domain completeness.
  • Application: if f is total, f is also partial; additional domain coverage is the distinguishing feature.

A database textbook shows a classification ladder: relation at the top, then partial function, then total function at the bottom.

  • Type: (X ⇸ Y) ⊆ (X ↔ Y) — every partial function is a relation.
  • Domain: a relation has no single-valued constraint; a partial function does.
  • Uniqueness: the relation allows multiple outputs per input; the function forbids it.
  • Application: (X → Y) ⊆ (X ⇸ Y) ⊆ (X ↔ Y) is the full chain.

An instructor draws the subset chain on a whiteboard: relation ⊇ partial function ⊇ total function.

  • Type: three levels of constraint, each stricter than the last.
  • Domain: relation — no domain requirement; partial — single-valued; total — single-valued + full domain.
  • Uniqueness: adding single-valuedness to a relation yields a function; adding full domain yields a total function.
  • Application: every total function is a partial function which is a relation — the chain never reverses.

V.22 Practice: Single-Valued Property

A contact list maps each person to phone numbers. Alice has two numbers: home and mobile.

Why is this a ↔ relation, not a ⇸ function?

Correct. Two outputs for one input breaks the single-valued property.

  • Type: ↔ allows multiple outputs; ⇸ allows at most one.
  • Domain: Alice is in the domain either way — that is not the issue.
  • Uniqueness: single-valued means ∀ x, y1, y2: if x ↦ y1 ∈ f ∧ x ↦ y2 ∈ f ⇒ y1 = y2.
  • Application: f(alice) would be ambiguous — which number?

8.2 Lambda Notation

I. Concept

  • Set comprehension form: f = {x : X | p • x ↦ e} — builds a function from a set description.
  • Lambda notation: (λ declaration | constraint • result) — a more concise alternative.
  • Constraint omission: when the domain is the whole source type, the constraint is omitted.
  • Definite description: μ x : S | p — the unique element of S satisfying p.
  • Characteristic tuple: (λ a : A; b : B; c : C • ...) has source A × B × C.
  • Left associativity: function application groups left — f g a = (f g) a.
  • Right associativity: the function arrow groups right — A → B → C = A → (B → C).

8.3 Functions on Relations

I. Concept

  • Relational operators as functions: domain restriction, range restriction, relational composition, relational inverse, and closure are all functions upon relations.
  • dom and ran: dom : (X↔Y) → ℙX; ran : (X↔Y) → ℙY.
  • ◁ / ▷: domain restriction A ◁ R keeps pairs whose first element is in A; range restriction R ▷ B keeps pairs whose second element is in B.
  • ⨟ (composition): R ⨟ S contains x ↦ z iff ∃ y with x ↦ y ∈ R and y ↦ z ∈ S.
  • ∼ (inverse): relational inverse R∼ swaps every pair — R∼ = { y ↦ x | x ↦ y ∈ R }.
  • Transitive closure R⁺: smallest transitive relation containing R; reflexive transitive closure R* adds id X.
  • Infix symbol convention: underscores mark argument positions, e.g., _ ◁ _.
  • Suffix symbol convention: single trailing underscore for postfix operators, e.g., _∼.

8.4 Overriding

I. Concept

  • Problem: f ∪ g may not be functional if dom f and dom g overlap — a single input could map to two outputs.
  • Overriding f ⊕ g: agrees with f outside dom g, but with g inside dom g — g wins on conflicts.
  • Formal definition: f ⊕ g = (dom g ◁ f) ∪ g — first remove conflicting entries from f via domain anti-restriction, then take union.
  • Domain formula: dom(f ⊕ g) = dom f ∪ dom g.
  • Disjoint domains: when dom f ∩ dom g = ∅, no conflict arises → f ⊕ g = f ∪ g = g ⊕ f (commutative).
  • Non-commutativity: in general f ⊕ g ≠ g ⊕ f because the second operand always wins on domain conflicts.
  • Single-entry update: f ⊕ {x ↦ v} replaces exactly one mapping in f.
  • Union equivalence: ⊕ reduces to ∪ exactly when domains are disjoint.

Two teams submit schedules: f = {alice ↦ mon, bob ↦ tue} and g = {carol ↦ wed, dave ↦ thu}. Their domains do not overlap.

Which statement is true?

  • Update: no conflicting keys exist.

Given dom f = {a, b, c} and dom g = {b, d}, an engineer asks for dom(f ⊕ g).

What is the result?

Correct. dom(f ⊕ g) = dom f ∪ dom g.

  • Update: dom g = {b, d}.
  • Override: override never shrinks the domain — it only replaces values.
  • Result: {a, b, c} ∪ {b, d} = {a, b, c, d}.

V.24 Practice: Non-Commutativity

Let f = {x ↦ 1} and g = {x ↦ 2}. A colleague claims f ⊕ g = g ⊕ f.

Is this correct?

Correct. The second operand wins, so swapping operands changes the result.

  • Original: f = {x ↦ 1}, g = {x ↦ 2}.
  • Update: dom f ∩ dom g = {x} ≠ ∅.
  • Override: f ⊕ g: g wins, so {x ↦ 2}. g ⊕ f: f wins, so {x ↦ 1}.
  • Result: f ⊕ g ≠ g ⊕ f when domains overlap.

V.25 Practice: Union Equivalence

An engineer asks: “Under what condition does f ⊕ g equal f ∪ g?”

Which condition is necessary and sufficient?

Correct. When domains are disjoint, there are no conflicts to resolve.

  • Original: f ⊕ g = (dom g ◁ f) ∪ g.
  • Update: if dom f ∩ dom g = ∅, then dom g ◁ f = f (nothing removed).
  • Override: f ⊕ g = f ∪ g.
  • Result: range disjointness is irrelevant; domain disjointness is the key condition.

8.5 Properties of Functions

I. Concept

  • Diverging arrows: characteristic of a function — no element maps to >1 target.
  • Converging arrows: two sources share one target → NOT injective.
  • Injection (one-to-one): ∀ x₁, x₂ : dom f • f(x₁) = f(x₂) ⇒ x₁ = x₂.
  • Surjection (onto): ran f = Y — every target element is reached.
  • Bijection (one-to-one correspondence): both injective and surjective.
  • Six arrows: partial/total × injection/surjection/bijection → ⤔, ↣, ⤀, ↠, partial bijection, ⤖.
  • Totality dimension: partial = dom f ⊆ X; total = dom f = X.
  • No converging arrows: necessary and sufficient for injectivity.

8.6 Finite Sets

I. Concept

  • Finite set: range of a total bijection from 1..n for some n ∈ ℕ.
  • Number range: m..n = {k : ℤ | m ≤ k ∧ k ≤ n}.

  • Countable: elements enumerable up to a bound n — core of finiteness.

  • Finite function: a function whose domain is a finite set — basis of finite sequences.
  • 𝔽 X: the set of all finite subsets of X (finite power set).
  • Foundation: finite sets underpin the theory of finite sequences in Z.
