Chapter 23 — A Save Area

23.1 Specification

I. Concept

  • Save area: a module with two operations — Save (push) and Restore (pop).
  • Abstract state: SaveArea == [ save_area : seq Record ] — a sequence used as a stack (LIFO).
  • Nondeterministic Save: Save ≅ Save0 ∨ SaveFullErr — may succeed (ok) or fail (full).
  • Why nondeterministic: free space depends on the rest of the system; modelling it would break modularity.
  • Loose constant n : ℕ: capacity parameter, instantiated during design.
  • Deterministic Restore: Restore ≅ Restore0 ∨ RestoreEmptyErr — disjuncts have disjoint preconditions (save_area ≠ ⟨⟩ vs save_area = ⟨⟩).
  • Error schemas use ΞSaveArea to leave state unchanged and emit a status report.

23.2 Design

I. Concept

  • First design decision: two-level memory — main (bounded) + secondary (unlimited).
  • Fix capacity: n : ℕ; n ≥ 1 — capacity parameter now bound.
  • Bounded sequence: bseq[X] = { s : seq X | #s ≤ n }.
  • Full sequence: fseq[X] = { s : seq X | #s = n }.
  • Concrete state: CSaveArea: main : bseq[Record], secondary : seq(fseq[Record]).
  • Retrieve: save_area = (⁀/ secondary) ⁀ main — total bijection.
  • Concrete Save: two disjuncts — #main = n (flush + seed) vs #main < n (append).
