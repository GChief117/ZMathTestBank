Chapter 3 — Predicate Logic

3.1 Predicate Calculus

I. Concept

  • A predicate is a statement with a gap — not yet true or false until the gap is filled.
  • Filling the gap with an object variable like x still leaves truth undetermined.
  • Adding a quantifier (∃ or ∀) turns a predicate into a full proposition.
  • Existential: ∃ x : ℕ • x > 5 — “there is a natural number greater than 5.”
  • Universal: ∀ x : ℕ • x > 5 — “every natural number is greater than 5” (false).
  • Generalised disjunction: ∃ ≈ checking value-by-value with ∨.
  • Generalised conjunction: ∀ ≈ checking value-by-value with ∧.
  • The language of predicate calculus reveals internal structure that propositional logic cannot express.

V.1 Practice: Predicate vs Proposition

Which best describes a predicate?

Correct. A predicate has a placeholder; filling it yields T or F.

  • Variable: putting x in the gap gives x > 5.
  • Quantify: ∃ x : ℕ • x > 5 is now a proposition.
  • Evaluate: true (e.g., 6 > 5).

V.21 Practice: Predicate Calculus Purpose

Why does predicate calculus extend propositional logic?

Correct. Predicate calculus lets us express “every” and “some.”

  • Predicate: statement with a gap.
  • Variable: fills the gap.
  • Quantify: ∀ / ∃ turn it into a proposition.
  • Evaluate: richer than bare propositional letters.

3.2 Quantifiers and Declarations

I. Concept

  • Z syntax: Q x : a | p • q — quantifier Q, bound variable x, range a, constraint p, predicate q.
  • Constraint role: (∃ x : a | p • q) ⇔ (∃ x : a • p ∧ q); (∀ x : a | p • q) ⇔ (∀ x : a • p ⇒ q).
  • Scope: bound variable’s scope = the constraint and predicate, up to the enclosing bracket.
  • Renaming: change a bound variable’s name without changing meaning, provided the new name is fresh.
  • Two same-quantifier declarations can merge with ; unless the first variable appears in the second’s range.
  • A variable not bound by any quantifier is free; the occurrence next to the quantifier symbol is a binding occurrence.
  • A variable can have both free and bound occurrences in the same expression.

3.3 Substitution

I. Concept

  • Substitution replaces each free occurrence of variable x in predicate p with a term y.
  • Notation: p[y / x] — result of substituting y for every free x in p.
  • Bound variables untouched: if x is bound by a quantifier, substitution does not affect it.
  • Variable capture: if the term being substituted has a free variable that would fall under a quantifier’s scope, the meaning changes unintentionally.
  • Fix capture: rename the bound variable to a fresh name before substituting.
  • Simultaneous substitution: p[t, u / x, y] replaces x and y at the same time — differs from sequential p[t / x][u / y].
  • Substitution distributes through all propositional operators: (¬p)[t/x] ⇔ ¬p[t/x], etc.

3.4 Universal Introduction and Elimination

I. Concept

  • Truth tables are useless for quantifiers — the sets bound variables range over are too large.
  • ∀-introduction: prove p for an arbitrary x ∈ a → conclude ∀ x : a • p.
  • Proviso: x must NOT be free in any undischarged assumption of the proof of p.
  • Full form with constraint: assume x ∈ a and p, prove q → conclude ∀ x : a | p • q.
  • ∀-elimination: from t ∈ a and ∀ x : a • p, conclude p[t / x].
  • Full form: from t ∈ a, ∀ x : a | p • q, and p[t / x], conclude q[t / x].
  • ∀ distributes through ∧: (∀ x : a • p ∧ q) ⇒ ((∀ x : a • p) ∧ (∀ x : a • q)).
  • If x not free in p: (∀ x : a • p ⇒ q) ⇔ (p ⇒ ∀ x : a • q).

3.5 Existential Introduction and Elimination

I. Concept

  • ∃-introduction: provide a witness t ∈ a with p[t/x] and q[t/x] → conclude ∃ x : a | p • q.
  • ∃-elimination: from ∃ x : a | p • q, introduce a fresh x, assume x ∈ a, p, and q, derive conclusion C (with x not free in C).
  • Witness name must not leak: the conclusion C must not mention the witness variable.
  • ∃ distributes through ∨: (∃ x : a • p ∨ q) ⇒ ((∃ x : a • p) ∨ (∃ x : a • q)).
  • One-point rule: ∃ x : a | x = t • q ⇔ q[t / x] — the unique witness t eliminates the quantifier.
