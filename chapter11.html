Chapter 11 — Schemas

11.1 The Schema

I. Concept

  • Two Z languages: mathematical language (objects & relationships) and schema language (structure & composition).
  • Schema: a structure introducing variables under constraint → declaration part + predicate part.
  • Horizontal form: [ declaration | predicate ] — brackets delimit, vertical bar separates.
  • Vertical (box) form: broken-box outline, horizontal bar separates declarations from predicate.
  • Naming: Name ≙ [ decls | pred ] or embed name in box top — creates abbreviation.
  • Schema equivalence: same variables + same constraints → same schema (hidden constraints in declarations count).
  • Multi-line readability: each decl on new line (omit ;), each conjunct on new line (omit ∧) — implicit conjunction.
  • Omitted predicate → true: all type-correct bindings accepted; no constraint imposed.

11.2 Schemas as Types

I. Concept

  • Fifth way to introduce a type: given set, free type, power set, Cartesian product — or schema type.
  • A binding assigns a value to each declared variable: ⟨ a ⇝ 2, c ⇝ {1,2,3} ⟩.
  • The schema type = set of all bindings satisfying the predicate.
  • Declare a variable of schema type: s : S means s is a valid binding of S.
  • Selection operator (_._): s.a denotes component a of binding s.
  • Differs from Cartesian product: components accessed by name, not position.
  • Schema types let you build composite types combining multiple different-typed fields under one name.

11.4 Schemas as Predicates

I. Concept

  • Schema as predicate: if every component of a schema has already been declared, the schema name introduces its predicate constraint.
  • Effect: declaration parts are discarded; only the constraints remain.
