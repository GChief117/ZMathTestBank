Chapter 13 — Promotion

13.1 Factoring Operations

I. Concept

  • Indexed components: large systems contain multiple instances of the same local state (database records, user accounts, array elements).
  • Uniform relationship: a global state maps indices to local states → changes to one component can be expressed locally.
  • Factoring: split a global op into a local operation + a promotion schema.
  • Separation of concerns: local op describes the effect; promotion schema describes the frame (which element, how global state updates).
  • Reconstitution: conjoin local op ∧ promotion, then hide local state via ∃ ΔLocal to get a global op.
  • Index types vary: partial function (game scores), relation (mail system), or sequence (data array) — factoring works for all.
  • Benefit: each factor can be specified and analysed in isolation; multiple local ops share one promotion schema.

13.2 Promotion

I. Concept

  • Formal definition: given Local, Global, LocalOperation, and a Promote schema → the local op is promoted to ∃ ΔLocal • Promote ∧ LocalOperation.
  • Promote schema contains: decorated/undecorated copies of both Local and Global, plus an index input (e.g., p? : Performance).
  • Promote predicates: (1) index ∈ domain; (2) extract local pre-state from global; (3) write local post-state back; (4) frame condition for non-target entries.
  • Functional index: when the global-to-local mapping is a function, existential quantification can be eliminated (one-point rule).
  • Non-promotable ops: operations on the global state itself (e.g., changing the set of announced performances) cannot be factored via promotion.
  • Totalisation: promoted ops are often partial; error schemas (using Ξ) handle cases like “not yet booking” and are disjoined to form total operations.

V.7 Practice: One-Point Rule

When the global-to-local mapping is a function, the existential quantifier in the promotion formula can be eliminated. This is called:

Correct. If θLocal = global(index?) uniquely determines the local state, ∃ can be removed.

  • Promote: functional index → unique binding.
  • One-point: ∃ x • x = t ∧ P(x) simplifies to P(t).
  • Result: the expanded schema has no quantifiers.

13.3 Free and Constrained Promotion

I. Concept

  • Free promotion: the Promote schema does not additionally constrain the local state — if an update is possible at all, it is possible for all local after-states.
  • Formal test: (∃ Local′ • ∃ Global′ • Promote) ⇒ (∀ Local′ • ∃ Global′ • Promote).
  • Meaning: the existential and universal quantifiers over Local′ can be exchanged — the promotion imposes no constraint beyond the local state invariant.
  • Constrained promotion: the global state invariant or promotion schema constrains which local after-states are possible, so the exchange fails.
  • Deciding freeness: expand ∃ Global′ • Promote; if θLocal′ appears unconstrained (beyond the local invariant), the promotion is free.
  • Rules used: existential elimination and universal introduction justify the quantifier exchange.
  • Trade-off: free promotions are more modular; constrained promotions may be the correct model when the global invariant genuinely restricts local behaviour.

The priority stack’s local state is PriData with priority : ℕ; data : Data. The global invariant orders elements by priority.
