Chapter 4 — Equality and Definite Description

4.1 Equality

I. Concept

  • Equality: e = f means e and f are identical (same object).
  • Reflection [eq-ref]: t = t — everything equals itself.
  • Leibniz’s law [eq-sub]: if s = t, substitute one for the other in any predicate.
  • Inequality: e ≠ f abbreviates ¬(e = f).
  • Symmetry [eq-symm]: if s = t then t = s — derived from [eq-ref] + [eq-sub].
  • Transitivity: if s = t and t = u then s = u — derived from [eq-sub].
  • Identity of Indiscernibles: s = t iff every property of s is a property of t.
  • Atomic propositions: equalities (with set membership) form the atoms of Z logic.

  • Apply: conclude p[s/x].
  • Conclude: same object, same predicates.

4.2 The One-Point Rule

I. Concept

  • One-point rule: eliminates a quantifier when the body pins x to a term via equality.
  • Existential form: ∃ x : a • p ∧ x = t ≡ t ∈ a ∧ p[t/x].
  • Universal form: ∀ x : a • x = t ⇒ p ≡ t ∈ a ⇒ p[t/x].
  • Proviso: x must not be free in t — otherwise t changes when x changes.
  • Derived rule: both directions proven from [eq-sub], [∃-intro], [∃-elim], [eq-symm].
  • Key benefit: removes a quantifier entirely, simplifying reasoning.
  • Common in specs: when a formula identifies a unique value, the quantifier is redundant.

II. Vocab Table

TermPlain English
One-point ruleQuantifier collapses when the body pins x to a fixed term.
Pinned variableA bound variable constrained by equality to a specific value.
Free variableA variable not bound by any quantifier in the expression.
Provisox must not be free in t for one-point to apply.
Derived ruleProvable from existing axioms; not primitive.
CollapseSimplification that removes the quantifier.

4.3 Uniqueness and Quantity

I. Concept

  • Equality enriches counting: phrases like “only,” “at most,” “at least,” “exactly” need = and ≠.
  • Only: P(x) ∧ ∀ y • P(y) ⇒ y = x.
  • At most one: ∀ x, y • P(x) ∧ P(y) ⇒ x = y.
  • At least two: ∃ x, y • x ≠ y ∧ P(x) ∧ P(y).
  • Unique quantifier ∃1: exactly one — combines existence and uniqueness.
  • Expansion: ∃1 x • P(x) ≡ ∃ x • P(x) ∧ (∀ y • P(y) ⇒ y = x).
  • Analogy: valid / satisfiable / unsatisfiable predicates ↔ tautologies / contingencies / contradictions.

4.4 Definite Description

I. Concept

  • Definite description: μ x : a • p — “the unique x in a such that p.”
  • Only meaningful when ∃1 x : a • p holds.
  • Undefined: if zero or many x satisfy p, the description has no denotation.

  • Purpose: name objects by their properties rather than by a given name.
  • Common in specs: “the admin,” “the current user,” “the CEO” — all definite descriptions.
