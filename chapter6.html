Chapter 6 — Definitions

6.1 Declarations

I. Concept

  • Purpose: a declaration introduces an object into a specification.
  • Basic type: write [Type] to introduce a basic type (given set) with opaque elements.
  • Variable: x : A introduces variable x drawn from set A.
  • Signature: x : t where t is a type → makes the type explicit.
  • Local constraint: x : t | x ∈ s — the constraint follows a vertical bar in set comprehensions, quantifications, or μ-expressions.
  • Global constant: if the constant is used throughout the spec → requires an axiomatic definition.
  • Non-ℤ sets: if the set is not ℤ, it must be defined elsewhere in the specification.

II. Vocab Table

SignatureA declaration x : t where t is a type.

6.2 Abbreviations

I. Concept

  • Form: symbol == term — introduces a new name as a synonym for an existing expression.
  • The new name (symbol) is a global constant with the same type and value as the term.
  • No recursion: the symbol being defined must NOT appear on the right — non-recursive.
  • Benign: asserts nothing new; can always be eliminated by substituting the RHS.
  • Inference rule [abbrev]: given s == e, conclude s = e.
  • Set abbreviation: constraining predicates must be properly introduced for the abbreviation to be well defined.

II. Vocab Table

TermPlain English
AbbreviationA definition symbol == term — a pure alias.
Symbol (LHS)The fresh name being introduced.
Term (RHS)The expression being named — must already be defined.
Non-recursiveThe LHS must not appear on the RHS.
BenignAsserts nothing — just a convenience.
EliminableCan be replaced by substituting the RHS everywhere.

III. Symbol Table

SymbolPlain English

∧Logical AND (in constraining predicates).
ℙPower set — set of all subsets.
{ … | … }Set comprehension with constraint.

6.4 Axiomatic Definitions

I. Concept

  • Form: a box with a declaration above a horizontal line and a predicate below.
  • The predicate is an axiom — assumed true whenever the symbol is used.
  • Constraining: the predicate may narrow the value to a unique object or leave it under-determined.
  • Global: introduces a constant available throughout the specification.
  • Example: ℕ : ℙ ℤ with ∀ z : ℤ • z ∈ ℕ ⇔ z ≥ 0 defines the natural numbers.
  • Consistency: the predicate must be satisfiable — an unsatisfiable axiom makes the spec inconsistent.
  • Multiple objects: a single axiomatic box can declare several objects at once.

  • Declare: one name per constant.
  • Constrain: redefinition is disallowed.
  • Axiom: unique binding.
  • Check: no clashes.

V.23 Practice: RGB Channels

A colour picker defines r, g, b each bounded by 255. Which axiomatic form captures this?

Correct. Three objects, conjunction of constraints.

  • Declare: r, g, b : ℕ.
  • Constrain: each ≤ 255.
  • Axiom: bounded channels.
  • Check: satisfiable — e.g. (0, 0, 0).

V.24 Practice: Function Declaration

Can an axiomatic definition declare a function?

Correct. The declaration may use any type.

  • Declare: e.g. f : ℤ → ℤ.
  • Constrain: predicate specifies behaviour.
  • Axiom: function behaviour is axiomatised.
  • Check: general definition mechanism.

V.25 Practice: Consistency Summary

The most critical requirement for an axiomatic definition is:

Correct. Unsatisfiable axiom = inconsistent spec.

  • Declare: constant exists.
  • Constrain: must have a witness.
  • Axiom: otherwise anything is provable.
  • Check: consistency is paramount.
