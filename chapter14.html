Chapter 14 — Preconditions

14.1 The Initialisation Theorem

I. Concept

  • Abstract data type → two proof opportunities: consistency and domain-checking.
  • Consistency: the state invariant must be satisfiable — at least one valid state exists.
  • Proved via the initialisation theorem: ∃ State′ • StateInit.
  • Decorated schema StateInit: describes the system right after initialisation.
  • Proof strategy: expand definitions → eliminate quantified variables via one-point rule.
  • If proof succeeds: spec is not vacuous — the data type has at least one reachable state.
  • Usually straightforward when the initial state is uniquely described by equations.

14.2 Precondition Investigation

I. Concept

  • Precondition: the states + inputs for which an operation is defined.
  • Formula: pre Op = ∃ State′ • Op \ outputs.
  • Hiding (\): removes after-state + output components → leaves before-state + inputs.
  • Surfaces hidden assumptions: constraints the spec depends on but may not state explicitly.
  • Partial operation: undefined when precondition fails → possible source of error.
  • Totalisation: disjoin error-case schemas to handle all violated preconditions.
  • Result of totalisation: pre TotalOp = true — every input handled.

  • Mechanical recipe for preconditions: straightforward and routine; tool-supportable.

14.4 Structure and Preconditions

I. Concept

  • Operation structure simplifies precondition calculation.
  • Distributivity over ∨: pre(Op1 ∨ Op2) = pre Op1 ∨ pre Op2.
  • Based on: ∃ A • P ∨ Q ⇔ (∃ A • P) ∨ (∃ A • Q).

  • Exception — no-contribution schema: if one conjunct has no before/input constraints, pre(Op ∧ S) = pre Op.
  • Free promotion: pre GOp = ∃ ΔLocal • pre LOp — global precondition reduces to local.
  • Constrained promotion: the simplification above does not apply.
