Chapter 20 — A Telecommunications Protocol

20.1 Specification: The External View

I. Concept

  • Case study: SS7 telecom protocol specified via abstraction + refinement.
  • External view: state = two sequences in, out : seq M — messages entered and messages delivered.
  • Suffix invariant: ∃ s : seq M • in = s ⁀ out — out is a trailing portion of in.
  • Orientation: newest messages added to the left; oldest toward the right. The gap between in and out represents in-flight messages.
  • ExtInit: ExtInit ≙ [ Ext′ | in′ = ⟨⟩ ] — forces both sequences empty.
  • Transmit: ΔExt; m? : M; in′ = ⟨m?⟩ ⁀ in; out′ = out — prepend new message; out unchanged.
  • Receive: ΔExt; in′ = in; #out′ = #out + 1 ∨ out′ = out — delivery grows by one or stays (nondeterministic).
  • Contract: all later design decisions must preserve this external behaviour — no re-ordering, no corruption.

20.2 Design: The Sectional View

I. Concept

  • Abstraction power: the full SS7 document is large; this Z spec is small because it abstracts away routing.
  • Sectional view: decomposes the channel into hops along a route of SPCs.
  • Section schema: route : iseq SPC; rec, ins, sent : seq(seq M) with invariants route ≠ ⟨⟩, #route = #rec = #ins = #sent, rec = ins ≈ sent, front sent = tail rec.
  • Pairwise concatenation (≈): (s ≈ t) i = (s i) ⁀ (t i) — combines per-section ins and sent into rec.
  • STransmit: new message enters at head rec (first section); sent unchanged.
  • SReceive: message delivered from last sent (last section); transfers from last ins.
  • Daemon: nondeterministically picks a section i with non-empty ins and moves oldest message to section i+1. Invisible externally.
  • Variant for liveness: an integer-valued variant could ensure messages eventually arrive (the spec as given does not insist on delivery).

20.3 Relationship Between External and Sectional Views

I. Concept

  • Goal: prove two views consistent — the sectional view correctly implements the external view.
  • Retrieve relation: head rec → in; last sent → out; distributed concat of ins → in-flight.
  • STransmit refines Transmit: appending at head rec matches appending to external in.
  • SReceive refines Receive: delivering from last sent matches consuming from external out.
  • Daemon refines ΞExt: internal hop is invisible externally — head rec and last sent unchanged.
  • SectionInit refines ExtInit: all-empty sections correspond to empty external state.
  • Inductive proof: base case (1-hop route trivial) + step (n → n+1 using front/last decomposition).
  • Sequence lemmas: head, last, front, tail distribute over pairwise concatenation.
